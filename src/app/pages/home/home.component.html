EN COURS DE DÉVELOPPEMENT Réflexions sur la partie Réservations

Pour le moment, l'essentiel de mon travail a porté sur l'implémentation de l'API, avec l'authentification, la gestion des rôles, les endpoint et les règles métiers concernant la création des lieux, des courts, des réservations, des participations...

Le frontend n'est pas encore beaucoup avancé. Il s'appuie sur le framework Angular 19 (hébergement OVH).

En backend, un serveur expose une API REST (conteneur Docker chez Render.com) développée avec Java Spring Boot (JDK 17). Les données sont stockées dans une base PostgreSQL.

J’ai entièrement construit le modèle de données : définition du MCD avec entités et relations, puis passage au MLD pour identifier les tables nécessaires. J'ai beaucoup aimé réfléchir à cela.

Plusieurs rôles définis :
  • User (joueur) : réserver un court (public/privé), ajouter des invités, rejoindre une réservation publique.
  • Manager (gestionnaire) : créer des lieux, associer des courts, gérer les réservations/participations de ses courts.
  • Admin : accès complet à toutes les fonctionnalités.

À l’aide de fichiers Dockerfile, un pipeline CI/CD a été mis en place pour automatiser (sur push main) le déploiement des conteneurs sur Render.com.
